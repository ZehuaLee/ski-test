Detailed Description

VirtIO console.

The VirtIO console uses the following queues:

  1. Port0 receive (from host to guest)
  2. Port0 transmit (from guest to host)
  3. Control receive (optional)
  4. Control transmit (optional)
  5. Port1 receive (optional)
  6. Port1 transmit (optional)

This implementation implements the minimum of features for one port, which means that the multiport feature is not announced and only queue 0 and 1 are available.

See also:
https://github.com/rustyrussell/virtio-spec
http://docs.oasis-open.org/virtio/virtio/v1.0/virtio-v1.0.html

Addr  cfgOffset

Read from the configuration space of a device.

This method is called by the transport interface to read data from a device model's configuration space. The device model should use the cfgOffset parameter as the offset into its configuration space.

Warning:
The address in the packet should not be used to determine the offset into a device's configuration space.

pkt Read request packet.
